Why "Save image as..." is disabled but "Copy image" works

Right-click on each of the images below to observe the context menu behavior.

1. CSS Background Images

If the picture is applied as a background style (background-image), the browser doesn’t expose it as a file element. You can copy the rendered pixels but not save the source directly.

2. Canvas-Drawn Images

When an image is drawn onto an HTML5 <canvas>, the browser only sees the canvas output. "Copy image" grabs the rendered bitmap, but "Save image as..." isn’t available because there’s no direct file for the drawn content.

3. Inline Encoded Images (Base64 or Data URI)

If the image is embedded inline (e.g., using a data: URI in the src attribute), Chrome may allow copying the rendered bitmap but not saving it as a separate file, as it's not a distinct file resource.
